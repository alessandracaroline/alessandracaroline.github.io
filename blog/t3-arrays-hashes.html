Ruby: Arrays and Hashes

A small comparison

June 28, 2015

Their common ground

Arrays and hashes are both containers of a collection of objects and thereby aid in storing and retrieving data. Arrays and hashes differ in the way the objects are ordered. Therefore, their usage


Arrays

Any object can be stored in an array. The objects stored in an array are ordered. Each has its slot, which is indexed by numbers starting with zero. The objects can be accessed through their index in the array. Arrays are best used when dealing with ordered lists.


Creating an array: 

        array = Array.new
        array = []

Example: 

        >> names = ["Alessandra","Caroline","Faisst"]
        => names = ["Alessandra","Caroline","Faisst"]
        >> names[1]
        => "Caroline"


Hashes

Hashes contain pairs of objects. They aren't assigned fixed indexed slots like objects in Arrays, but any object can refer to a slot. In a pair of object, one is the key and the other represents the value. The value can be accessed through the key. Keys are unique per hash. Any object can represent the hash key or value, but there can only be one value associated with any given key.


Creating a hash: 

        array = Hash.new
        array = {}

Example: 

        >> names = {
                  "first_name" => "Alessandra",
                  "middle_name" => "Caroline",
                  "last_name" => "Faisst"
                  }

        >> names["middle_name"]
        => "Caroline"
